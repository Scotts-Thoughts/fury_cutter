## Project Overview

Fury Cutter detects and timestamps Pokemon trainer battles in gameplay videos, outputting JSON metadata used to automate labeling in Adobe Premiere Pro. Supports 11 game versions across Generations 1–5 (Black, Platinum, HeartGold, Emerald, Ruby, Sapphire, FireRed, LeafGreen, Crystal, Yellow, Red).

## Running

```bash
# Main analysis (single file or batch folder)
python fury_cutter.py <video_or_folder> --version <game> [options]

# GUI (or double-click "Fury Cutter.bat")
python fury_cutter_gui.py

# Apply labels to Premiere Pro
python premiere_label_sender.py <json_file>

# Extract templates from screenshots
python extract_templates.py
```

There are no formal tests, linting, or build steps. The `test_*.py` files are ad-hoc OCR verification scripts, not a test suite.

## Architecture

### Detection Pipeline (fury_cutter.py)

Processing flows through three stages:

1. **Black/white frame detection** — Scans the video at configurable intervals, looking for transition frames (mean pixel < 5 = black, > 250 = white) that mark battle boundaries.
2. **Trainer identification** — At each detected battle region, identifies the trainer using one of two methods:
   - **Template matching** (preferred, 100–350x faster): `cv2.matchTemplate()` against pre-extracted grayscale PNGs in `templates/{game}/`. Threshold 0.9 (0.95 for "champion").
   - **OCR fallback** (Tesseract): Generation-dependent preprocessing. Gen1–4 use percentile-based thresholding; Gen5 uses raw frames. Matches against two patterns: `"[name]'s Team"` or `"Leader [name]"`.
3. **Boundary refinement** — Binary search backward/forward from the detection point to find exact cut-in and cut-out frames.

When OBS chapter markers exist in the MP4 metadata, `extract_obs_chapters.py` provides battle timestamps directly, skipping the full video scan.

### Game Configuration

All game-specific parameters live in `GAME_CONFIGS` (fury_cutter.py): OCR crop regions, gameplay regions, trainer lists, platform, generation, and OCR pattern type. `TEMPLATE_FOLDER_MAP` maps game versions that share overlays to the same template directory (e.g., Ruby/Sapphire → emerald templates).

### Template System

`templates/{game}/` directories contain grayscale PNG crops of trainer battle headers plus a `metadata.json` mapping filenames to canonical trainer names. Generated by `extract_templates.py` from screenshots placed in `templates/matching/{game}/`.

### Premiere Pro Integration

Output JSON maps each battle to a Premiere label type via `PREMIERE_LABELS` (Rival, Gym, E4, Champion, etc.). `premiere_label_sender.py` drives Premiere via `pyautogui` keyboard shortcuts. The `.jsx` scripts are ExtendScript alternatives for direct Premiere scripting.

### Key Constants

- Frame thresholds: `BLACK_MEAN_THRESHOLD=5`, `WHITE_MEAN_THRESHOLD=250`
- Default intervals assume 240fps video: `--transition-jump=720` (3s), `--early-interval=480` (2s), `--normal-interval=1440` (6s)
- Template match threshold: 0.9 general, 0.95 for "champion"

## Dependencies

Requires Tesseract OCR installed at `C:\Program Files\Tesseract-OCR\tesseract.exe`. Python packages: opencv-python, numpy, pytesseract, Pillow. Premiere automation additionally needs pyautogui, pygetwindow.
